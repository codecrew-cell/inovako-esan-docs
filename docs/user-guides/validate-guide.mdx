Validate ekrani, Run sonucunda uretilen detection ve OCR verilerinin insan
tarafindan kontrol edildigi ana kalite kapisidir. Bu ekrandaki duzeltmeler
Lithology ve Export ciktisini dogrudan etkiler.

<Warning>
  Bu ekran bir kalite kapisidir: burada birakilan hata, Lithology ve Export
  ciktilarina dogrudan tasinir.
</Warning>

## Ana ekran bolumleri

| Bolum | Gorev |
| --- | --- |
| Image viewer | Aktif karot goruntusunu ve detection kutularini gosterir. |
| Maneuver table | Derinlik araliklari ve self-edit uyarilarini listeler. |
| Header navigation | Onceki/sonraki goruntu ve sayfa gecislerini yonetir. |
| Canvas tools | Kutu secme, duzenleme, silme ve goruntu tipi secimi saglar. |

## Adim adim akis

<Steps>
  <Step title="Detection'lari incele">
    Aktif goruntudeki kutu konumlarini ve siniflarini kontrol edin.

    <Frame caption="Detection canvas">
      ![Validate detections](/assets/screens/validate-detections-canvas.png)
    </Frame>
  </Step>
  <Step title="Kutu duzelt / sinif degistir">
    Eksik kutu ekleyin, hatali kutuyu silin veya sinifini degistirin.

    <Frame caption="Kutu duzenleme ve sinif degistirme akisi">
      ![Validate box edit](/assets/screens/validate-box-edit-class-change-flow.png)
    </Frame>
    <Frame caption="Sinif degisikligi">
      ![Validate class change](/assets/screens/validate-class-change-new.png)
    </Frame>
  </Step>
  <Step title="Manevra tablosunu kontrol et">
    OCR kaynakli derinlik hatalarini ve `selfEdit` uyarilarini duzeltin.

    <Frame caption="Uyari iceren manevra tablosu">
      ![Validate maneuver warning](/assets/screens/validate-warning-maneuver-table.png)

    Tum degisiklikleri tek save ile gonderin, ardindan Lithology veya Export'a gecin.
  </Step>
</Steps>

## Kritik endpointler

| Endpoint | Ne zaman kullanilir |
| --- | --- |
| `GET /maneuvers_from_changes/{session_id}` | Changes tablosundan manevra uretmek icin. |
| `GET /maneuversPhoto/{session_id}` | Manevra gorsellerini olusturmak icin. |
| `GET /get_maneuvers/{session_id}` | Manevra tablosunu okumak icin. |
| `GET /getColorChange/{hole_id}` | Renk degisimi bilgisini almak icin. |
| `GET /frame/{session_id}` | Aktif goruntu frame'ini yuklemek icin. |
| `POST /save_bulk_changes/{session_id}` | Kullanici degisikliklerini toplu kaydetmek icin. |

## Engelleyici kontroller

Validate ekraninda `selfEdit` degerleri kullanilarak OCR/manevra hatalari
isaretlenir. Esit veya ters derinlikler export oncesi duzeltilmelidir.

| Kontrol | Beklenen durum |
| --- | --- |
| Manevra baslangici | Bitis degerinden kucuk olmali. |
| Bos manevra tablosu | Run veya changes verisi kontrol edilmeli. |
| Kutu-row uyumu | Detection kutulari ilgili karot satiri icinde kalmali. |
| Takip eden ekranlar | Lithology ve export icin session verisi kaydedilmis olmali. |

## Kaydetme standardi

<Tip>
  - Toplu kaydetme yapmadan sayfadan ayrilmayin.
  - Birden fazla kutu/tablo degisikligi yapildiysa tek save ile backend'e gonderin.
  - Kayit sonrasi manevra tablosunu tekrar kontrol edin.
  - Export oncesi minimum bir goruntu ve bir manevra satiri manuel incelenmelidir.
</Tip>
