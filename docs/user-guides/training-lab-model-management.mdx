Training Lab sadece sinif duzenleme ekrani degildir; ayni zamanda egitilen
modellerin kaydedildigi, indirildigi, yeniden adlandirildigi ve veri
versiyonlariyla iliskilendirildigi model yonetim alanidir.

<Frame caption="Model egitimi">
  ![Model training](/assets/screens/training-lab-model-train-new.png)
</Frame>

## Model yasam dongusu

<Steps>
  <Step title="Feature space hazirlanir">
    Dataset, composition veya tile snapshot kaynak olarak secilir.
  </Step>
  <Step title="Siniflar temizlenir">
    Cluster adlari, lasso secimleri, exclude islemleri ve merge kararlari uygulanir.
  </Step>
  <Step title="Atamalar finalize edilir">
    `/traininglab/finalize-assignments` veya ilgili UI aksiyonu ile aktif etiket seti olusturulur.
  </Step>
  <Step title="Egitim baslatilir">
    `/traininglab/train-stream` SSE akisi egitim ilerlemesini canli verir.
  </Step>
  <Step title="Model kaydedilir">
    `/traininglab/save-model` model artifact kaydini olusturur.
  </Step>
  <Step title="Lineage baglanir">
    Model, kullanilan snapshot veya composition bilgisiyle Data Versioning tarafina kaydedilir.
  </Step>
</Steps>

## Model islemleri

| Islem | Endpoint | Aciklama |
| --- | --- | --- |
| Model listeleme | `GET /traininglab/models` | Kayitli modelleri ve metadata bilgisini listeler. |
| Yeniden adlandirma | `POST /traininglab/rename-model` | Model gorunen adini degistirir. |
| Silme | `POST /traininglab/delete-model` | Model kaydini kaldirir veya kullanilamaz hale getirir. |
| Kaydetme | `POST /traininglab/save-model` | Son egitilen modeli kalici kayda alir. |
| Model indirme | `GET /traininglab/download-model` | Model dosyasini indirir. |
| Encoder indirme | `GET /traininglab/download-encoder` | Feature encoder dosyasini indirir. |
| Gecmis | `GET /traininglab/training-history` | Egitim gecmisi listesini verir. |
| CSV gecmis | `GET /traininglab/training-history.csv` | Egitim gecmisini raporlama icin CSV verir. |

## Kayit oncesi kalite kontrol

<Warning>
  Asagidaki maddeler tamamlanmadan model kaydetmeyin; eksik bilgi ileride modelin
  hangi veriyle egitildigini izlemeyi zorlastirir.
</Warning>

<Check>Cluster isimleri varsayilan `Cluster 0`, `Cluster 1` gibi kalmadi.</Check>
<Check>Exclude edilen siniflar ve ornekler son kez kontrol edildi.</Check>
<Check>Merge sonrasi sinif sayisi hedef sinif setiyle uyumlu.</Check>
<Check>Dogru tile snapshot version secildi; draft tile aktifse Data Cleanout'ta ana tile netlestirildi.</Check>
<Check>Model adinda proje, veri versiyonu ve tarih bilgisi var.</Check>
