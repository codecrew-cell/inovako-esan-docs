Dashboard, kullanicinin sisteme girdikten sonra platform durumunu ve son analiz
ozetlerini tek ekranda gormesi icin tasarlanmistir. Giris ekrani kullanici adini
dogrular, yeni bir tarayici oturumu baslatir ve kullaniciyi Run ekranina yonlendirir.

## Giris akisi

<Steps>
  <Step title="Kullanici adini gir">
    Kullanici adinizi girin ve giris yapin.

    <Frame caption="Giris ekrani">
      ![Login](/assets/screens/login.png)
    </Frame>
  </Step>
  <Step title="Dogrulama">
    Sistem `GET /ldap_auth/{username}` istegi ile kullaniciyi dogrular.
  </Step>
  <Step title="Oturum olusturma">
    Dogrulama basariliysa tarayici icin yeni bir `login_session_id` uretilir ve
    `POST /sessions/start` ile backend tarafinda oturum kaydedilir.
  </Step>
  <Step title="Run ekranina gecis">
    Uygulama Run ekranina gecer ve eski analiz cache durumlarini temizler.
  </Step>
</Steps>

<Note>
  Giris sirasinda uygulama `sessionStorage` icinde `login_username` ve
  `login_session_id` degerlerini saklar. Bu degerler performans loglari, Gantt
  gorunumu, hata izleme ve oturum kurtarma akislariyla iliskilidir.
</Note>

## Dashboard kartlari

Dashboard Start ekraninda `GET /dashboard/stats` verisi kullanilir. Backend cevap
veremezse ekran guvenli varsayilan degerlerle acilir.

| Alan | Ne anlatir |
| --- | --- |
| `total_core_length` | Islenmis toplam karot uzunlugu. |
| `recovery_rate` | Kurtarim yuzdesi. |
| `class_counts` | Litoloji veya sinif sayimlari. |
| `fault_zones` | Tespit edilen fay/ozel bolge sayisi. |
| `active_models` | Kullanilabilir aktif model sayisi. |
| `ai_confidence` | Son analizlerden uretilen genel guven metrigi. |
| `system_status` | Sistem servislerinin ozet durumu. |
| `recent_scans` | Son tarama/analiz kayitlari. |
| `timeline_items` | Son operasyon hareketleri. |
| `map_sites` | Lokasyon kartlari ve saha harita verisi. |

## Konum bilgisi

<Tip>
  Tarayici izin verirse Dashboard, kullanicinin yaklasik konumunu gostermek icin
  browser geolocation bilgisini kullanir. Adres metni icin OpenStreetMap Nominatim
  reverse geocoding cagirilir. Konum izni verilmezse ekran `N/A` veya varsayilan
  lokasyonla calisir.
</Tip>

## Operasyon notlari

- Giris butonuna tekrar tekrar basilmasi engellenir; aktif istek varsa yeni istek baslatilmaz.
- Giris sirasinda onceki Redux analiz state'i ve arayuz cache kayitlari temizlenir.
- `BASE_URL` tanimli degilse frontend varsayilan olarak `http://localhost:8002` adresini kullanir.
- Dashboard verileri karar destek amaclidir; kesin proje kaydi icin ilgili analiz ekranindaki ciktilar ve export dosyalari esas alinmalidir.
