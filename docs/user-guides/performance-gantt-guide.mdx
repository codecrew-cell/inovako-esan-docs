Performance ekrani son run isleminin zaman dagilimini ve isleme adimlarini Gantt
mantiginda gosterir. Amac yavaslayan model, OCR veya dosya isleme adimini hizli
tespit etmektir.

<Info>
  Bu ekrana `canRun` yetkisi olan kullanicilar erisebilir (`/performance`).
</Info>

## Veri kaynagi

| Endpoint | Amac |
| --- | --- |
| `GET /gantt/latest` | En son session icin Gantt verisini getirir. |
| `GET /gantt/{session_id}` | Belirli session icin Gantt verisini getirir. |

Backend tarafinda timing verisi isleme sirasinda yazilir ve model/worker bazli
sureleri icerir.

## Kullanici akisi

<Steps>
  <Step title="Run tamamla">
    Run ekraninda bir islem tamamlayin.
  </Step>
  <Step title="Performance'i ac">
    Performance ekranini acin ve latest Gantt verisinin geldigini kontrol edin.
  </Step>
  <Step title="Yavas adimi bul">
    `Models` veya diger detay filtreleriyle yavas adimi tespit edin.
  </Step>
  <Step title="Kaynaga in">
    Sorun model bazliysa Settings/model dosyasini, OCR bazliysa backend loglarini kontrol edin.
  </Step>
</Steps>

## Yorumlama

<AccordionGroup>
  <Accordion title="Tek model adimi asiri uzun" icon="gauge">
    GPU/CPU, model dosyasi veya TensorRT fallback kontrol edilmeli.
  </Accordion>
  <Accordion title="OCR adimlari uzun" icon="gauge">
    Goruntu kalitesi, satir sayisi veya OCR worker performansi incelenmeli.
  </Accordion>
  <Accordion title="Ilk batch uzun, sonraki batch normal" icon="gauge">
    Model warm-up veya cache etkisi olabilir.
  </Accordion>
  <Accordion title="Tum batch'ler yavas" icon="gauge">
    Donanim, batch size veya disk I/O problemi olabilir.
  </Accordion>
</AccordionGroup>

## Operasyon notlari

<Note>
  - Gantt ekrani hata ayiklama aracidir; kullanici raporu yerine performans analizi icindir.
  - Release oncesi ayni test dataset ile sure karsilastirmasi yapilmalidir.
  - Settings `batch_size`, model tipi ve `PREFER_TRT` davranisi performansi etkileyebilir.
</Note>
