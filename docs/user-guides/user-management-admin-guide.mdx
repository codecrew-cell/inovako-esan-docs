User Management ekrani sadece admin kullanicilar icindir. Buradan lokal kullanici
kayitlari, roller ve aktiflik durumu yonetilir.

<Info>
  Bu ekrana yalnizca `admin` rolu erisebilir (`/user-management`). Mutating
  endpoint'ler `X-Username` header'i ile gelen kullanicinin admin olup olmadigini
  kontrol eder.
</Info>

## Islem listesi

| Islem | Endpoint | Not |
| --- | --- | --- |
| Kullanici listele | `GET /users` | Tum lokal kullanicilari listeler. |
| Kullanici ekle | `POST /users` | Username, rol ve aktiflik bilgisi kaydeder. |
| Rol degistir | `PATCH /users/{username}/role` | Admin kontrolu gerekir. |
| Aktif/pasif yap | `PATCH /users/{username}/active` | Pasif kullanici login akisini gecmemelidir. |
| Kullanici sil | `DELETE /users/{username}` | Geri alinmasi manuel dosya restore gerektirir. |

## Operasyon kurallari

<Warning>
  - Admin sayisini gereksiz artirmayin.
  - Pasif yapma islemi, silmeye gore daha guvenli ilk adimdir.
  - Rol degisikligi sonrasi kullanicinin tarayici oturumu yenilenmelidir.
  - Public dokumanda gercek kullanici listesi veya mail adresi paylasilmaz.
</Warning>

## Hata durumlari

<AccordionGroup>
  <Accordion title="403 / unauthorized" icon="triangle-alert">
    **Neden:** `X-Username` admin degil.

    **Cozum:** Aktif admin hesabiyla giris yapin.
  </Accordion>
  <Accordion title="Kullanici gorunmuyor" icon="triangle-alert">
    **Neden:** `users.json` okunamiyor veya backend farkli dizinde.

    **Cozum:** Backend calisma dizinini kontrol edin.
  </Accordion>
  <Accordion title="Rol degisti ama menu degismedi" icon="triangle-alert">
    **Neden:** Frontend session state eski.

    **Cozum:** Logout/login veya sayfa yenileme yapin.
  </Accordion>
</AccordionGroup>
