# AutoML Guide

AutoML ekrani klasik siniflandirma/takoz modeli egitimi, model kaydi, model aciklamasi ve test islemleri icin kullanilir. Training Lab daha yeni embedding tabanli akisi sunar; AutoML ise mevcut legacy model yonetimiyle calisir.

## Yetki

| Route | Guard |
| --- | --- |
| `/automl` | `canRun` |

## Ana kullanim senaryolari

| Senaryo | Aciklama |
| --- | --- |
| Dataset yukleme | Egitim verisi klasoru AutoML backend'e yuklenir. |
| Model egitme | `cls` veya `takoz` modunda egitim baslatilir. |
| Model ismi kaydetme | Yeni model adi ve aciklamasi model listesine yazilir. |
| Model test etme | Secili model tekil ornek uzerinde denenir. |
| Model silme / aciklama guncelleme | Model registry temizligi yapilir. |

## Endpointler

| Endpoint | Amac |
| --- | --- |
| `POST /automl_upload_folder/` | Egitim klasoru yukler. |
| `GET /train` | Secili parametrelerle egitimi baslatir. |
| `GET /checkClsModelNames` | Classification model listesini getirir. |
| `GET /saveClsModelName/{name}` | Yeni model adini ve aciklamasini kaydeder. |
| `POST /changeClsModelDescriptions` | Model aciklamasini degistirir. |
| `POST /deleteClsModel` | Secili modeli listeden/diskten siler. |
| `GET /test_classification` | Secili modeli test goruntusu uzerinde calistirir. |

## Egitim oncesi kontrol

- Dataset sinif klasorleri dogru isimlendirilmis olmali.
- Model adi benzersiz olmali.
- Training mode (`cls` veya `takoz`) dogru secilmeli.
- Settings tarafindaki AutoML parametreleri kontrol edilmeli.
- Egitim sonrasi modelin Run/Settings tarafinda gorunup gorunmedigi test edilmeli.

## Training Lab ile farki

| Alan | AutoML | Training Lab |
| --- | --- | --- |
| Temel yaklasim | Legacy model egitimi | Embedding, UMAP, class cleaning ve model lineage |
| Veri kaynagi | Yuklenen klasor | Folder, Data Platform snapshot veya tile snapshot |
| Model kaydi | Classification model listesi | Training Lab registry + Data Platform lineage |
| Test | `test_classification` | Training Lab Test tab |
