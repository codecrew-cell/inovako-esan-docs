Validate is the main quality gate where the detection and OCR data produced by Run
is reviewed by a human. The corrections made on this screen directly affect the
Lithology and Export output.

<Warning>
  This screen is a quality gate: any error left here carries directly into the
  Lithology and Export outputs.
</Warning>

<Frame caption="Main canvas with detection boxes">
  ![Validate detections canvas](/assets/screens/validate-detections-canvas.png)
</Frame>

## What it does

<CardGroup cols={2}>
  <Card title="Review detections" icon="scan-search">
    Box positions, classes and image variants are reviewed.
  </Card>
  <Card title="Edit boxes" icon="pen-line">
    Add a missing box, delete an incorrect box or change its class.
  </Card>
  <Card title="Maneuver table" icon="table">
    OCR-sourced start/end depths are corrected.
  </Card>
  <Card title="Bulk save" icon="save">
    Session changes are written to the backend in one go.
  </Card>
</CardGroup>

## Main screen sections

| Section | Role |
| --- | --- |
| Image viewer | Shows the active core image and detection boxes. |
| Maneuver table | Lists depth ranges and self-edit warnings. |
| Header navigation | Manages previous/next image and page transitions. |
| Canvas tools | Box selection, editing, deletion and image type selection. |

## Core concepts

| Concept | Description |
| --- | --- |
| Core (Karot) | Cylindrical rock samples extracted underground during drilling. All images belong to these. |
| Spacer (Takoz) | A marker showing the depth range in core boxes; carries metre info (e.g. 12.30-13.50) and is used as a reference point. |
| Solid Core | Intact core keeping its cylindrical form; high-quality rock structure. |
| Broken Core | Fragmented/broken core; medium structural weakness. |
| Fault Zone | Crushed, disintegrated rock zone; the weakest and most problematic zones. |
| Fracture (major) | A prominent, deep break along the core. Old name `Fracture_A`. |
| Joint (minor) | A surface/small crack. Old name `Fracture_B`. |

## Canvas interactions

<AccordionGroup>
  <Accordion title="Zoom / Pan" icon="move">
    Mouse wheel zooms in/out centered on the cursor; left-click drag pans (cursor becomes "grab"). Zoom in/out/reset and the current zoom percentage are at the bottom right.
  </Accordion>
  <Accordion title="Box selection and resizing" icon="pen-line">
    The selected box is highlighted; if it is not a spacer, left/right handles appear. Only **horizontal** resizing is possible; height and vertical position are fixed. While a box is selected, pan is disabled and a "Pan Locked" warning appears.
  </Accordion>
  <Accordion title="Original / Processed mode" icon="images">
    Switch between the raw image (Original) and the model-processed version (Processed) from the right panel. The boxes stay the same in both modes; only the background changes.
  </Accordion>
  <Accordion title="OCR depth editing" icon="text-cursor-input">
    Double-click the black OCR label next to a spacer to edit the value; Enter saves, Escape cancels. A depth change re-computes the related maneuver rows.
  </Accordion>
  <Accordion title="Adding a new box" icon="plus">
    The "+" in the right panel opens box-add mode; the cursor becomes a crosshair. Click and drag on the photo (a red dashed area) to draw a new box. The class selector on the left sets the new box's class; the trash button deletes the selected box.
  </Accordion>
</AccordionGroup>

## Level filters

In the right panel, maneuvers are filtered by severity level. At least one level
must stay active; if a level has no maneuvers it cannot be selected.

<CardGroup cols={3}>
  <Card title="Level 1" icon="circle-check">
    Problem-free maneuvers.
  </Card>
  <Card title="Level 2" icon="circle-alert">
    Medium-level cases with certain conditions.
  </Card>
  <Card title="Level 3" icon="octagon-alert">
    Serious errors; should be reviewed first.
  </Card>
</CardGroup>

The maneuver card shows the selected maneuver's order (colored badge), the problem
type, the depth range and its position within the total (progress bar). The arrow
buttons navigate only among maneuvers matching the active filter.

## Maneuver table columns

The table at the bottom of the screen shows, numerically, all geotechnical analysis
results for the currently displayed core photo. Each row represents a specific
depth interval (maneuver) and all of its metrics are presented already computed.

<Info>
  **The table is fully dynamic.** When you add a box, delete one, change its class
  or update the spacer depth in the image area, the table is recomputed instantly.
  Except for `Core_Diameter`, data is not edited directly in the table; all changes
  are made through the image area. The table is always bound to the currently open
  photo and refreshes automatically when you switch images.
</Info>

| Column | Meaning | Formula |
| --- | --- | --- |
| `Hole_ID` | Drillhole identifier/code; shows which hole the row belongs to. | — |
| `Depth_From` | Start depth (m) of the interval. | — |
| `Depth_To` | End depth (m) of the interval. | — |
| `Interval_Length` | Maneuver length. | `Depth_To - Depth_From` |
| `Recovery_Interval` (TCR) | Total (recovered) core length between two spacers. | — |
| `Recovery_pct` (TCR %) | Total core recovery percentage. | `(Recovery_Interval / Interval_Length) * 100` |
| `RQD_Interval` | Total length of solid core pieces longer than 10 cm. | — |
| `RQD_pct` | Rock quality percentage. | `(RQD_Interval / Interval_Length) * 100` |
| `Solid_Recovery_Interval` (SCR) | Total length of pieces keeping their cylindrical (solid core) form. | — |
| `Solid_Recovery_pct` (SCR %) | Solid core recovery percentage. | `(Solid_Recovery_Interval / Interval_Length) * 100` |
| `Fracture_Count` | Total number of fractures/cracks counted in the interval. | — |
| `Broken_Core_Interval` | Total length of heavily fragmented core (fracture system indistinguishable). | — |
| `Fault_Zone_Interval` | Total length of crushed/disintegrated zones that lost rock integrity. | — |
| `Core_Diameter` | Core diameter/type info; can be entered manually by the user. | — |
| `Logged_Date` | Record/log date. | — |
| `Logged_By` | User who logged/ran the analysis. | — |
| `Self_Edit` | System-detected quality/warning info (e.g. missing core blocks, ocr error). Shows which rows to prioritize. | — |

### Key formulas

```text
Interval_Length = Depth_To - Depth_From
TCR_pct = (TCR_Interval / Interval_Length) * 100
RQD_pct = (RQD_Interval / Interval_Length) * 100
SCR_pct = (SCR_Interval / Interval_Length) * 100
```

### Abbreviations

| Abbreviation | Expansion |
| --- | --- |
| RQD | Rock Quality Designation |
| TCR | Total Core Recovery |
| SCR | Solid Core Recovery |
| OCR | Optical Character Recognition |
| ID | Identifier |
| Pct / % | Percent |
| From / To | Start / End |
| Interval | Range between two depth points |
| Core | Core (karot) |
| Fault Zone | Crushed/disintegrated rock zone |

<Tip>
  The structural-damage metrics (`Fracture_Count`, `Broken_Core_Interval`,
  `Fault_Zone_Interval`), evaluated together, summarize how solid or problematic
  the rock structure is. Rows with a `Self_Edit` warning should be reviewed first.
</Tip>

## Critical endpoints

| Endpoint | When it is used |
| --- | --- |
| `GET /maneuvers_from_changes/{session_id}` | To generate maneuvers from the changes table. |
| `GET /maneuversPhoto/{session_id}` | To create maneuver images. |
| `GET /get_maneuvers/{session_id}` | To read the maneuver table. |
| `GET /getColorChange/{hole_id}` | To get color change info. |
| `GET /frame/{session_id}` | To load the active image frame. |
| `POST /save_bulk_changes/{session_id}` | To bulk save user changes. |

## Blocking checks

On the Validate screen, `selfEdit` values are used to flag OCR/maneuver errors.
Equal or reversed depths must be corrected before export.

| Check | Expected state |
| --- | --- |
| Maneuver start | Must be smaller than the end value. |
| Empty maneuver table | The Run or changes data should be checked. |
| Box-row consistency | Detection boxes should stay within the relevant core row. |
| Downstream screens | Session data must be saved for lithology and export. |

## Save standard

<Tip>
  - Do not leave the page without doing a bulk save.
  - If multiple box/table changes were made, send them to the backend with a single save.
  - After saving, recheck the maneuver table.
  - Before export, at least one image and one maneuver row should be reviewed manually.
</Tip>
