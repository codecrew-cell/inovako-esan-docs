Training Lab is not just a class-editing screen; it is also the model management
area where trained models are saved, downloaded, renamed and linked to data
versions.

  ![Model training](/assets/screens/training-lab-model-train-new.png)
</Frame>

## Model lifecycle

<Steps>
  <Step title="Feature space is prepared">
    A dataset, composition or tile snapshot is selected as the source.
  </Step>
  <Step title="Classes are cleaned">
    Cluster names, lasso selections, exclude operations and merge decisions are applied.
  </Step>
  <Step title="Assignments are finalized">
    The active label set is created via `/traininglab/finalize-assignments` or the related UI action.
  </Step>
  <Step title="Training is started">
    The `/traininglab/train-stream` SSE stream provides live training progress.
  </Step>
  <Step title="Model is saved">
    `/traininglab/save-model` creates the model artifact record.
  </Step>
  <Step title="Lineage is linked">
    The model is recorded on the Data Versioning side with the snapshot or composition used.
  </Step>
</Steps>

## Model operations

| Operation | Endpoint | Description |
| --- | --- | --- |
| List models | `GET /traininglab/models` | Lists saved models and metadata. |
| Rename | `POST /traininglab/rename-model` | Changes the model display name. |
| Delete | `POST /traininglab/delete-model` | Removes the model record or makes it unusable. |
| Save | `POST /traininglab/save-model` | Persists the last trained model. |
| Download model | `GET /traininglab/download-model` | Downloads the model file. |
| Download encoder | `GET /traininglab/download-encoder` | Downloads the feature encoder file. |
| History | `GET /traininglab/training-history` | Returns the training history list. |
| CSV history | `GET /traininglab/training-history.csv` | Returns training history as CSV for reporting. |

## Pre-save quality check

<Warning>
  Do not save a model before completing the items below; missing info makes it
  harder to later trace which data the model was trained on.
</Warning>

<Check>Cluster names are not left as defaults like `Cluster 0`, `Cluster 1`.</Check>
<Check>Excluded classes and samples were checked one last time.</Check>
<Check>After merges, the class count matches the target class set.</Check>
<Check>The correct tile snapshot version is selected; if a draft tile is active, the main tile was finalized in Data Cleanout.</Check>
<Check>The model name includes project, data version and date info.</Check>

## Best practice for lineage

Before saving a model, check the active source info:

<CardGroup cols={3}>
  <Card title="Folder" icon="folder-open">
    Dataset Key must be entered and the dataset snapshot record completed.
  </Card>
  <Card title="Composition" icon="layers">
    The composition must be in committed state.
  </Card>
  <Card title="Tiles" icon="grid-2x2">
    The correct tile snapshot version must be selected.
  </Card>
</CardGroup>
