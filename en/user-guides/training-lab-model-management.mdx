## Purpose

Training Lab is not just a class-editing screen; it is also the model
management area where trained models are saved, downloaded, renamed and linked
to data versions.

## Model lifecycle

1. **Feature space is prepared:** A dataset, composition or tile snapshot is selected as the source.
2. **Classes are cleaned:** Cluster names, lasso selections, exclude operations and merge decisions are applied.
3. **Assignments are finalized:** The active label set is created via `/traininglab/finalize-assignments` or the related UI action.
4. **Training is started:** The `/traininglab/train-stream` SSE stream provides live training progress.
5. **Model is saved:** `/traininglab/save-model` creates the model artifact record.
6. **Lineage is linked:** The model is recorded on the Data Versioning side with the snapshot or composition used.

## Model operations

| Operation | Endpoint | Description |
| --- | --- | --- |
| List models | `GET /traininglab/models` | Lists saved models and metadata. |
| Rename | `POST /traininglab/rename-model` | Changes the model display name. |
| Delete | `POST /traininglab/delete-model` | Removes the model record or makes it unusable. |
| Save | `POST /traininglab/save-model` | Persists the last trained model. |
| Download model | `GET /traininglab/download-model` | Downloads the model file. |
| Download encoder | `GET /traininglab/download-encoder` | Downloads the feature encoder file. |
| History | `GET /traininglab/training-history` | Returns the training history list. |
| CSV history | `GET /traininglab/training-history.csv` | Returns training history as CSV for reporting. |

## Pre-save quality check

- Cluster names should not be left as defaults like `Cluster 0`, `Cluster 1`.
- Excluded classes and samples should be checked one last time.
- After merge operations, the class count should match the expected target class set.
- The tile snapshot selection must be the correct version; if a draft tile is active, the main tile record should be finalized on the Data Cleanout side.
- It is recommended that the model name include project, data version and date info.

## Best practice for lineage

Before saving a model, check the active source info:

| Source | Check |
| --- | --- |
| Folder | Dataset Key must be entered and the dataset snapshot record completed. |
| Composition | The composition must be in committed state. |
| Tiles | The correct tile snapshot version must be selected. |

If this info is missing, even if the model file is produced, it becomes harder
to later trace which data it was trained on.
