Run is the main entry point for image analysis. It is where raw core images are
loaded into the system and the main analysis job is started. The Validate,
Lithology and Export flows do not work properly until this screen is completed.

<Info>
  The user must have the `run` permission to access this screen.
</Info>

<Frame caption="Run screen overview">
  ![Run screen overview](/assets/screens/run-overview.png)
</Frame>

## What it does

<CardGroup cols={2}>
  <Card title="Load a dataset" icon="folder-open">
    The images in the well folder are registered with the backend.
  </Card>
  <Card title="Validate file names" icon="spell-check">
    Folder and file names are checked against the well/depth standard.
  </Card>
  <Card title="Select models" icon="layers">
    The main YOLO model, spacer model and classification model are selected.
  </Card>
  <Card title="Analysis and progress" icon="activity">
    Backend workers run the image, OCR, row and detection steps; status is tracked via `/progress/`.
  </Card>
</CardGroup>

## Prerequisites

<Note>
  - The backend must be running.
  - The frontend `BASE_URL` must point to the correct backend address.
  - Model files must be present under `Model/Model Files`.
  - Folder/file names must match the expected well standard.
</Note>

## Step by step

<Steps>
  <Step title="Select and validate the folder">

    well/depth standard; non-conforming names are shown as a warning before processing.

## File naming convention

For a correct and stable flow on the Run screen, file names must follow a standard
format. The system derives the ordering from the BOX and metre-range info in the
file name.

```text
<Hole_ID>_BOX<boxNo>&<partNo>_<StartMetre>-<EndMetre>.<ext>
```

Example files:

```text
OZD-219_BOX01&02_0.00-7.00.jpeg
OZD-219_BOX03&04_7.00-12.00.jpeg
OZD-219_BOX05&06_12.00-20.30.jpeg
OZD-219_BOX07&08_20.30-28.50.jpeg
```

The system reads the following from these names:

| Part | Example | Meaning |
| --- | --- | --- |
| Hole_ID | `OZD-219` | Drillhole/well code |
| BOX range | `BOX01&02` | Box range the image covers |
| Metre range | `0.00-7.00` | Depth range of the image |

<Note>
  Images are processed not in upload order but in **ascending order** based on the
  box/metre info in the file name:
  `0.00-7.00 -> 7.00-12.00 -> 12.00-20.30 -> 20.30-28.50`.
</Note>

## Model selection

The Run screen fetches the model lists from the backend.

| List | Endpoint | Note |
| --- | --- | --- |
| Main model | `GET /checkMainModelNames` | Segmentation/detection model. |
| Spacer model | `GET /checkTakozModelNames` | Used for spacer detection. |
| Classification model | `GET /checkClsModelNames` | Classification support model. |

<Tip>
  The default models selected on the Settings screen may be applied automatically on the Run screen. Verify the model name is correct before the operation.
</Tip>

## File validation

| Check | Endpoint | If it fails |
| --- | --- | --- |
| Folder/file standard | `GET /validate_filenames/?folder_name=...` | An error is shown to the user before the folder is processed. |
| Image list | `GET /get_images/{folder_name}` | If the image list is empty, processing should not be started. |
| Static image access | `/static/uploaded_data/...` | If the thumbnail/preview fails to load, the path is checked. |

## Success criteria

<Check>The progress value advances and the backend returns no error.</Check>

## Common errors

<AccordionGroup>
  <Accordion title="File name validation error" icon="triangle-alert">
    **Likely cause:** The folder name and the image prefix do not match.

    **Fix:** Correct the file names according to the well standard.
  </Accordion>
  <Accordion title="Model list is empty" icon="triangle-alert">
    **Likely cause:** The model folders are missing or the backend cannot scan models.

    **Fix:** Check `Model/Model Files` and the Settings model names.
  </Accordion>
  <Accordion title="Progress does not advance" icon="triangle-alert">
    **Likely cause:** A backend worker is stuck or there is a GPU/CPU problem.

    **Fix:** Check the backend logs and the Error Tracking screen.
  </Accordion>
  <Accordion title="Validate opens but has no data" icon="triangle-alert">
    **Likely cause:** The session record was not completed.

    **Fix:** Retry the run and check the backend response.
  </Accordion>
</AccordionGroup>
