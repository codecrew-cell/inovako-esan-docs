# Run Screen Guide

## Purpose

Run is the main entry point for image analysis. Operators load a folder, choose models, start processing, and then move to validation screens.

## Basic workflow

1. Select or upload a core image folder.
2. Confirm that the image list is visible.
3. Select the required models.
4. Start processing.
5. Watch progress until completion.
6. Continue to Validate.

## Key controls

| Control | Purpose |
| --- | --- |
| Folder selection | Loads the source images for the active well/session. |
| Model selectors | Choose detection, block, OCR, or classification models. |
| Process button | Starts backend image processing. |
| Progress panel | Shows current processing state. |
| Navigation actions | Move to Validate or related screens after processing. |

## Common checks

- The folder contains the expected images.
- Backend is running before processing starts.
- Model files are available in the configured model directory.
- Progress reaches completion before switching screens.
