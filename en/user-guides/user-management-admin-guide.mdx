The User Management screen is for admin users only. Local user records, roles and
activity status are managed here.

<Info>
  Only the `admin` role can access this screen (`/user-management`). Mutating
  endpoints check whether the user coming via the `X-Username` header is an admin.
</Info>

## Roles

<CardGroup cols={3}>
  <Card title="admin" icon="shield">
    Authorized user who manages users and the system.
  </Card>
  <Card title="supervisor" icon="user-cog">
    User who manages operation settings and the control process.
  </Card>
  <Card title="engineer" icon="user">
    User who works on the Run, Validate, Training Lab and Data flows.
  </Card>
</CardGroup>

## Operation list

| Operation | Endpoint | Note |
| --- | --- | --- |
| List users | `GET /users` | Lists all local users. |
| Add user | `POST /users` | Saves username, role and activity info. |
| Change role | `PATCH /users/{username}/role` | Requires admin check. |
| Activate/deactivate | `PATCH /users/{username}/active` | A passive user must not pass the login flow. |
| Delete user | `DELETE /users/{username}` | Undo requires a manual file restore. |

## Operation rules

<Warning>
  - Do not unnecessarily increase the number of admins.
  - Deactivating is a safer first step than deleting.
  - After a role change, the user's browser session should be refreshed.
  - Do not share a real user list or email addresses in public documentation.
</Warning>

## Error cases

<AccordionGroup>
  <Accordion title="403 / unauthorized" icon="triangle-alert">
    **Cause:** `X-Username` is not an admin.

    **Fix:** Log in with an active admin account.
  </Accordion>
  <Accordion title="User not visible" icon="triangle-alert">
    **Cause:** `users.json` cannot be read or the backend is in a different directory.

    **Fix:** Check the backend working directory.
  </Accordion>
  <Accordion title="Role changed but menu did not" icon="triangle-alert">
    **Cause:** The frontend session state is stale.

    **Fix:** Logout/login or refresh the page.
  </Accordion>
</AccordionGroup>
