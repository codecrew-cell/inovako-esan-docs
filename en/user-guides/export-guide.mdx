The Export screen turns the data produced by the Validate and Lithology flows into
reportable images and tables. The final files shared with users are obtained from
this screen.

<Note>
  **Prerequisites:** Run must be completed, Validate changes saved, and mineral
  maneuvers generated. If lithology is used, the Lithology editor output must be saved.
</Note>

## Main outputs

| Output | Source | Check |
| --- | --- | --- |
| Maneuver images | `ManueverBlocks` static folder | Are the hole id and depth ranges correct? |
| Combined image of all files | Backend export folder | Are the image order and rows consistent? |
| CLS combined image | Classification output | Are the class colors as expected? |
| Color change image | Color analysis | Are the color transitions reasonable? |
| ZIP download | Frontend JSZip | Are all expected files in the package? |

## Step by step

<Steps>
  <Step title="Prepare the maneuver data">
    If `get_mineral_maneuvers` returns no data, it is regenerated via `mineral_from_changes`.
  </Step>
  <Step title="Check the preview">
    Verify the maneuver images and table against the hole id / depth ranges.
  </Step>
  <Step title="Save the maneuvers">
    Persist the output with `save_maneuvers`.
  </Step>
  <Step title="Download the ZIP">
    All images and Excel tables are downloaded as a ZIP.
  </Step>
</Steps>

## Endpoints

| Endpoint | Purpose |
| --- | --- |
| `GET /get_mineral_maneuvers/{session_id}` | Reads mineral maneuver data for export. |
| `POST /mineral_from_changes/{session_id}` | Regenerates missing mineral maneuvers. |
| `POST /getManueverImageNames` | Lists the maneuver images to download. |
| `GET /download/{hole_id}/{file}` | Downloads a single image file. |
| `GET /save_maneuvers/{session_id}` | Persists the maneuver output. |
| `/ManueverBlocks/...` | Serves the combined export images statically. |
| `GET /litho/image` | Serves the lithology source image for preview. |

## User checklist

<Check>The hole id is correct.</Check>
<Check>Maneuver start/end values are consistent.</Check>
<Check>Geotechnical, mineral and lithology output belong to the same session.</Check>
<Check>The row order in combined images is intact.</Check>
<Check>After downloading the ZIP, the file names and count are correct.</Check>

## Quality gate

Since the Export screen is the report generation point, an error seen here usually
comes from a previous step.

<AccordionGroup>
  <Accordion title="Export data is empty" icon="triangle-alert">
    Go back to the Validate/Lithology flow and regenerate the maneuvers.
  </Accordion>
  <Accordion title="Depth range is wrong" icon="triangle-alert">
    Return to the Validate table and correct the OCR/depth values.
  </Accordion>
  <Accordion title="Lithology color/region is wrong" icon="triangle-alert">
    Return to the Lithology editor and correct the segments.
  </Accordion>
  <Accordion title="No file is produced" icon="triangle-alert">
    Check the backend `ManueverBlocks` and log folders.
  </Accordion>
</AccordionGroup>
